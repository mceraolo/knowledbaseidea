Something seems to be wrong with either the PD² Adapter, PDS, GFEBS or SPS Legacy Integration (SPSLI). No documents are being extracted from PD². Also, no documents are being created in PD². What information should I provide the SPS Help Desk to assist in troubleshooting?


  1. The results of the Adapter Buffer Diagnostic (by date range) script in Script-Aid.
  2. The results of the Event Subscription Diagnostic script in Script-Aid.
  3. The results of the Retrieve Document Information script in Script-Aid for any documents that should have been extracted from the PD²
  4. A screen capture of Partner Profiles in My webMethods (Administration > Integration > B2B> Partner Administration > Partner Profiles)
  5. A screen capture of Transactions in My webMethods (Monitoring > Integration > B2B > Transactions > Saved tab). Ensure that you run a query first by clicking the Search button. (See KB ID 12005)
  6. A copy of the most webMethods error, server, and MWS logs (see KB ID 7028).
  7. The following file from your PD² Adapter server: C:\SoftwareAG\IntegrationServer\instances\default\bin\server.bat
  8. The following file from your PD² Adapter server: C:\SoftwareAG\IntegrationServer\instances\default\config\jdbc\pool\MWS Pool.xml
  9. The following file from your PD² Adapter Server: C:\SoftwareAG\MWS\server\default\config\mws.db.xml
